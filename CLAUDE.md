开发一个跨设备的无线语音输入系统，包含前后端和客户端，完成开发并测试每个组件都能够正常运行。

利用手机 Web 端捕获语音输入法生成的文本及修正过程，通过中继服务器，实时注入到 Windows 电脑的当前光标处。

现在的开发环境 windows 中可以使用 sudo 命令。

1. Mobile Web 端 (React)
提供带有输入历史等功能的精美 UI，还能有哪些功能你自己想一想，界面UI务必使用液态玻璃的风格，就是 apple ios 26 发布的那种液态玻璃效果，在网页端实现。应该是由这种开源项目的。
必须精准捕获输入框的文本变化及输入法的 composition（拼音/修正）事件，以适配语音输入法边吐字边修改的动态特性。本地长期维护一个匿名 DeviceID。提供扫码功能，用于读取 PC 端的连接信息。若当前设备被其他手机抢占导致断开连接，UI 界面需给予清晰提示。

2. 中继服务端 (Go)
负责 WebSocket 转发与动态 RoomToken 管理。
必须实现强制抢占逻辑：当新设备（DeviceID）携带有效的 RoomToken 扫码接入时，立刻主动断开旧设备的连接并转移控制权。
部署要求： 服务端需打包成 Docker 容器，提供一键部署的配置。

3. Windows 客户端 (C# .NET WPF/WinForms)
系统托盘常驻应用，需提供精美的交互界面，点击可呼出或刷新包含 RoomToken 的二维码。建立 WebSocket 客户端监听服务端指令，并将接收到的文本和退格动作实时且安全地注入到当前电脑的光标处。

所有的信息交换都应该端对端加密，参考 signal。
