## 问题 1: Windows 客户端更新检查失败

### 根因分析

仓库已是 public 的，Velopack 的 `GithubSource` 不需要 token。RELEASES 文件格式也正确：
```
D321FC771D2607CBB448C389B73610FFD512D629 Fluentia-1.7.6-full.nupkg 81652406
```

**真正原因：Velopack 版本过旧。** 项目使用 `0.0.1053`（pre-release），最新稳定版是 `1.2.0`。
0.0.x 系列的 `GithubSource` 实现存在已知问题：
- 可能无法正确解析 GitHub API 的 release 响应
- 可能无法正确处理 GitHub 的 asset 下载重定向
- 可能存在版本比较逻辑 bug

### 修复方案

**升级 Velopack 到 1.2.0。** API 兼容，改动很小：

**文件**: `client/Fluentia/Fluentia.csproj`
```xml
<!-- 旧 -->
<PackageReference Include="Velopack" Version="0.0.1053" />
<!-- 新 -->
<PackageReference Include="Velopack" Version="1.2.0" />
```

**文件**: `client/Fluentia/MainWindow.xaml.cs` — `GetUpdateManager()` 方法
```csharp
// 0.0.x 和 1.x 的 GithubSource 构造函数兼容
private UpdateManager GetUpdateManager()
{
    _updateManager ??= new UpdateManager(new GithubSource(
        "https://github.com/wxyu71/Fluentia", null, false));
    return _updateManager;
}
```

**文件**: `client/Fluentia/App.xaml.cs` — VelopackApp 初始化
```csharp
// 0.0.x: VelopackApp.Build().Run();
// 1.x:   VelopackApp.Build().Run();  // API 不变
```

如果升级后仍有问题，添加调试日志：
```csharp
private async Task ManualCheckForUpdatesAsync()
{
    ...
    var info = await mgr.CheckForUpdatesAsync();
    System.Diagnostics.Debug.WriteLine($"Velopack CheckForUpdates result: {info?.TargetFullRelease?.Version ?? "null"}");
    ...
}
```

### 改动文件

| 文件 | 改动 |
|------|------|
| `client/Fluentia/Fluentia.csproj` | Velopack 0.0.1053 → 1.2.0 |

## 问题 4: 浏览器客户端 PWA 自动更新

### 当前状态

项目已有 `vite-plugin-pwa` 配置:
- `registerType: 'autoUpdate'`
- `clientsClaim: true`
- `skipWaiting: true`
- `cleanupOutdatedCaches: true`
- 服务端已设置 `Cache-Control: no-cache, must-revalidate`

### 为什么自动更新仍然失败

Service Worker 的更新检查依赖浏览器的启发式策略（通常每 24 小时或导航时检查一次）。但问题在于:

1. **SW 缓存了 `index.html`**，其中引用了带 hash 的 JS bundle
2. 新版本部署后，旧 SW 仍服务缓存的 `index.html` → 引用的 JS bundle 已不存在 → 页面可能直接出错
3. 虽然 `skipWaiting: true` 让新 SW 立即激活，但浏览器可能根本不检查 SW 更新

### 修复方案: 版本轮询 + 自动刷新

在 App.tsx 中添加版本轮询机制：每 5 分钟请求 `/api/config`，比较服务器版本与本地 `__APP_VERSION__`，不一致时自动刷新。

**文件**: `mobile/src/App.tsx`

```tsx
// 版本轮询 — 检测服务器新版本后自动刷新
useEffect(() => {
  const poll = () => {
    const info = loadStoredConnection();
    if (!info) return;
    
    const httpBase = info.s.replace(/\/ws.*$/, '').replace('wss://', 'https://').replace('ws://', 'http://');
    fetch(`${httpBase}/api/config`, { cache: 'no-store' })
      .then(r => r.json())
      .then(config => {
        if (config.version && config.version !== APP_VERSION) {
          // 清除所有 SW 缓存并刷新
          window.caches?.keys()?.then((names: string[]) => 
            Promise.all(names.map((n: string) => window.caches.delete(n)))
          ).then(() => window.location.reload());
        }
      })
      .catch(() => {/* silent */});
  };
  
  // 首次检查延迟 30 秒（等连接稳定），之后每 5 分钟一次
  const initial = setTimeout(poll, 30_000);
  const interval = setInterval(poll, 5 * 60_000);
  return () => { clearTimeout(initial); clearInterval(interval); };
}, []);
```

同时在 visibility change 时也触发检查：

```tsx
useEffect(() => {
  const handleVisibility = () => {
    if (document.visibilityState === 'visible') {
      // 页面可见时立即检查版本
      checkServerVersion();
    }
  };
  document.addEventListener('visibilitychange', handleVisibility);
  return () => document.removeEventListener('visibilitychange', handleVisibility);
}, []);
```
